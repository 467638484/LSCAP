srsRAN环境搭建教程：

1、安装 Ubuntu 16.04 LTS 最新版的实体机环境

2、安装配置 Ubuntu 系统
   
   1）系统设置走一圈(隐私，显示，更新)，打开终端(禁用回滚行数限制，设置透明)
   
   2）连接网络，"设置-软件和更新-下载自"选择其他站点，选择mirrors.tuna.tsinghua.edu.cn，点击选择服务器
   
   3）sudo apt-get update 然后 sudo apt update
   
   4）将目录路径更改为中文
      export LANG=en_US
	  xdg-user-dirs-gtk-update
	  选择update names(不勾选下次不要询问我)
	  export LANG=zh_CN
	  重启
	  勾选不要再次询问我，选择保留旧的名称
   
   5）更换低延迟内核
      sudo apt-get update
	  sudo apt update
      sudo apt-get install linux-image-lowlatency linux-headers-lowlatency
	  sudo update-grub
	  重启
	  
   6）开机选择进入Ubuntu高级选项中的低延迟内核(不是upstart，不是recovery mode，是lowlatency的第一个)
      使用uname -a和uname -r进行验证
   
   7）清除多余内核
      查看内核
      dpkg --get-selections | grep linux
      清除内核
	  sudo apt-get purge xxx
	  其中对于generic而言，一般有5个，都需要清除，看列表中与通用内核版本号一致的一并清除就行(这里是把名字中带4.15.0-112的都清除掉，名字可以叠加到一条命令中，中间用空格隔开)
	  更新grub文件
	  sudo update-grub
	  发现又安装了4.15.0-142的通用内核，所以需要重复以上2个步骤，将4.15.0-142的内核清除，最后查看内核中没有通用内核即可
	  最后再次更新grub文件
	  sudo update-grub
	  
   8）修改Ubuntu的grub引导的启动顺序(默认启动项)
      查看/boot/grub/grub.cfg中低延迟内核的序号，选中menuentry，使用ctrl+f进行搜索，注意menu和submenu的区别，在外层标号从0开始往后加1，menu若是x，则其后面的submenu是x+1，submenu内部的标号又从0开始往后加1，内外层独立
      在/etc/default/grub中修改(修改之前使用sudo cp grub grub.bak保存一个备份)
	  将GRUB_DEFAULT=0修改为GRUB_DEFAULT="1> 0"
	  这是根据上面查看的序号决定的，该低延迟内核位于外层列表第2个位置中内层子列表的第1个位置，因为内外层标号均从0开始，所以为1> 0，注意>和0之间有一个空格
	  其实不改也行，则开机后默认选中的就是第一个的"Ubuntu"，其默认指向的是第二个的"Ubuntu高级选项"中的第一个
	  更新grub文件
	  sudo update-grub
	  重启验证
   
   9）安装完整语言包支持
      系统设置-语言支持-完整安装
   
   10）软件更新器，更新全部
       重启，开机后选择进入外层第二个的"Ubuntu高级选项"中的低延迟内核，这是为了防止更新过程中又重新安装了通用内核(一般不会再安装了)，若发现又安装了通用内核，则还需要按照上述步骤进行清理和修改启动顺序，清理虽然不是必须的，但最好这样做，以避免人为操作失误使程序运行在通用内核(应该运行在低延迟内核)，这样做也比较干净
	   
   11）sudo apt update
   
   12）配置浏览器设置
   
   13）安装net-tools
   
   14）安装trash-cli
   
   15）安装gnutls-bin
   
   16）安装git
   
   17）安装视频编解码支持
       sudo apt-get install ubuntu-restricted-extras
   
   18）安装GParted
   
   19）安装wireshark
   
   20）安装cmake
   
   21）重启
   
3、部署srsRAN平台
   
   1）安装electron-ssr-0.2.6.AppImage
      将该文件拷入目标实体机的Ubuntu系统，右键Properties(属性)，Permissions(权限)勾选Allow executing file as program(允许作为程序执行文件)
	  https://www.cnblogs.com/zienzir/p/10612399.html
	  双击文件图标(或右键选择运行)，点击下载ssr软件，下载完成后进入节点配置操作界面
	  配置相应的节点信息，点击确定
	  选择系统代理模式为全局代理
	  打开系统设置-网络-网络代理，切换为手动，并按照软件界面（点击服务器-订阅管理-通用设置）的相应信息
      配置http代理和https代理（127.0.0.1:12333），socks主机代理（127.0.0.1:1080）
      点击应用到整个系统
	  配置完成，以后每次使用时，运行electron-ssr-0.2.6.AppImage，然后打开系统设置-网络-网络代理，切换为手动的上述配置，点击应用到整个系统，
	  使用结束后，点击图标选择退出，然后打开系统设置-网络-网络代理，切换为无，点击应用到整个系统
   
   2）安装UHD射频前端驱动
      先关闭electron-ssr-0.2.6.AppImage
      sudo add-apt-repository ppa:ettusresearch/uhd
	  再打开electron-ssr-0.2.6.AppImage
	  sudo apt-get update
	  sudo apt-get install libuhd-dev libuhd3.15.0 uhd-host
	  下载安装完成后执行uhd_find_devices或uhd_usrp_probe命令，根据提示，运行命令sudo /usr/lib/uhd/utils/uhd_images_downloader.py
   
   3）安装packages
	  sudo apt-get install libpcsclite-dev pcscd pcsc-tools
	  sudo apt-get install libdw-dev
   
   4）安装srsgui
      sudo apt-get install libboost-system-dev libboost-test-dev libboost-thread-dev libqwt-qt5-dev qtbase5-dev
	  git clone -b release_2_0_qt5 https://github.com/srsLTE/srsGUI.git
      cd srsGUI
      mkdir build
      cd build
      cmake ../
      make
	  make test
	  sudo make install
	  sudo ldconfig
   
   5）安装srsran
      sudo apt-get install build-essential cmake libfftw3-dev libmbedtls-dev libboost-program-options-dev libconfig++-dev libsctp-dev
	  git clone -b release_21_10 https://github.com/srsRAN/srsRAN.git
      cd srsRAN
      mkdir build
      cd build
      cmake ../
      make
      make test
      sudo make install
      sudo srsran_install_configs.sh user
      sudo ldconfig

   6）重启
